- 라우터 -
네트워크를 통과하는 *데이터 패킷(Data Packet)*의 목적지 주소를 추출하여 가장 적절한 경로를 설정하고, 그 경로에 따라 패킷을 다음 장치로 전송
    **데이터 패킷**
        큰 데이터를 나누어 보내는 일정량의 전송 단위
  
- 게이트웨이 -
게이트웨이는 서로 다른 네트워크(예: LAN과 WAN, 또는 서로 다른 프로토콜의 네트워크)를 연결해 데이터를 전달하는 장치 또는 기능

- IP -
네트워크에서 장치를 식별하는 숫자 주소입니다.
  
- TCP -
전송 프로토콜로 데이터 분할 (패킷으로 나눔), 순서 보장, 손실 시 재전송, 오류 검출을 하는 프로토콜입니다.

- Domain Name -
도메인 네임은 네트워크에서 사용하는 IP 주소를 사람이 이해하기 쉬운 문자 형태로 표현한 주소이며, DNS(Domain Name System) 서버를 통해 해당 도메인과 IP 주소가 매핑되어 관리된다.

- 여러 관련 명령어 -
$ ip addr : 장치의 IP 주소 확인
$ nslookup 호스트명(Domain name) : 호스트명에 대한 IP주소를 알려준다.
$ finger 사용자명 : 사용자에 대한 자세한 정보를 알려준다.

- 파일 잠금 -
한 프로세스가 그 영역을 읽거나 수정할 때 다른 프로세스의 접근을 제한, 만약 제한하지 않을 시 원치 않은 결과가 생길 가능성이 생긴다.

& 파일 잠금 관련 명령어
   flock(<파일>, <커멘드>)
                  ㄴLOCK_SH : 공유 잠금으로 한개 이상의 프로세스들이 파일에 대한 공유 잠금이 가능 
                  ㄴLOCK_EX : 배타 잠금으로 한번에 하나의 프로세스만이 파일에 대한 배타 잠금을 할 수 있고, 만약 파일에 이미 배타 잠금이 걸려 있으면 기다림
                  ㄴLOCK_UN : 파일에 걸려 있는 잠금을 해제함
                  ㄴLOCK_NB : 파일에 잠금이 걸려 있으면 기다리지 않고 반환함

- 레코드 잠금 -
 파일의 일부 영역에 대해서 잠금한다.

& 레코드 잠금 관련 명령어                     
      int fcntl(<파일>, <커멘드>, struct flock *lock)
                        ㄴF_SETLK  : 잠금 설정    ㄴ 잠금 정보 구조체
                        ㄴF_SETLKW : 잠금 설정 
                        ㄴF_GETLK  : 잠금 정보 조회
