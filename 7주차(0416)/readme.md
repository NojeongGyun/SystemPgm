<pre>
- i-node -
파일 크기, 파일 소유자, 권한(rwx), 생성/수정 시간, 데이터 블록 위치, 링크 수 같은 파일의 정보와 데이터 위치를 저장합니다. 
무조권적으로 파일을 생성하면, 그 파일은 하나의 i-node가 할당됩니다.

- 데이터 블록 - 
저장한 파일의 내용이 담겨져 있습니다. 만약 아무것도 없이 파일만 만들어져있다면, 데이터 블록에 할당되지 않을 수도 있습니다.

저장할 크기보다 블록노드1개의 크기보다 크면 그 넘어가는 크기만큼의 블록노드의 개수가 할당이 됩니다.
ㄴ직접 블록 : inode 안에 데이터 블록 주소가 직접 저장하는 구조이며 작은 파일을 저장하기에 용이하고, 접근이 가장 빠름
ㄴ간접 블록 : inode가 데이터 블록을 직접 가리키지 않고, 블록 주소들을 저장한 블록을 가르키는 구조이며, 큰 파일을 저장할 때 용이하지만, 접근 속도가 느림
ㄴ이중 간접 블록 : inode가 데이터 블록을 직접 가리키지 않고, 블록 주소들을 저장한 블록을 가르키고, 한번 더 가르키는 구조이며, 매우 큰 파일을 저장할 떄 용이하지만, 접근 속도가 매우 느림

파일 시스템은 파일 크기에 따라: 직접 블록 → 간접 블록 → 이중 간접 블록 순으로 확장하여 사용합니다.
